# Build Mall 요구사항 문서

## 1. 프로젝트 개요

### 1.1 프로젝트 목적
공사 자재를 온라인에서 판매하기 위한 B2B/B2C 쇼핑몰 플랫폼 구축

### 1.2 대상 고객
- 건설업체 및 공사 현장 담당자
- 인테리어 업체
- 개인 소비자 (DIY 및 소규모 공사)
- 도매/소매 업체

## 2. 핵심 기능 요구사항

### 2.1 사용자 관리 (User Management)

#### 2.1.1 회원가입 및 인증
- 이메일/전화번호 기반 회원가입
- 소셜 로그인 (선택 사항)
- 이메일 인증
- SMS 인증 (선택 사항)
- 비밀번호 재설정
- 회원 정보 수정
- 회원 탈퇴

#### 2.1.2 사용자 유형
- **일반 사용자**: 개인 소비자
- **기업 사용자**: 건설업체, 인테리어 업체 등 (사업자 등록증 검증 필요)
- **관리자**: 시스템 관리자

#### 2.1.3 권한 관리
- 역할 기반 접근 제어 (RBAC)
- 사용자별 권한 설정
- 기업 사용자: 대량 주문, 계약 가격, 신용 구매 등

### 2.2 상품 관리 (Product Management)

#### 2.2.1 상품 카테고리
- 계층적 카테고리 구조 (대분류 > 중분류 > 소분류)
- 카테고리별 상품 필터링
- 주요 카테고리 예시:
  - 시멘트/콘크리트
  - 철근/철골
  - 벽돌/블록
  - 타일/석재
  - 목재/합판
  - 방수/단열재
  - 도배/페인트
  - 전기/조명
  - 수도/배관
  - 건축자재 기타

#### 2.2.2 상품 정보
- 상품명, 설명, 상세 이미지
- 상품 사양 (규격, 재질, 제조사 등)
- 기술 자료 및 시공 가이드 (PDF)
- 사용 가능 지역 정보
- 배송 정보 (무게, 크기, 배송 제약사항)

#### 2.2.3 상품 가격
- 기본 판매가
- 기업 사용자 할인가
- 대량 구매 할인 (수량별 차등 할인)
- 계약 가격 (기업별 협상 가격)
- 가격 이력 관리

#### 2.2.4 재고 관리
- 실시간 재고 현황
- 재고 부족 알림
- 입고 예정일 표시
- 안전 재고 설정
- 재고 이력 관리

#### 2.2.5 상품 검색 및 필터링
- 키워드 검색
- 카테고리별 필터
- 가격 범위 필터
- 재고 여부 필터
- 제조사별 필터
- 규격별 필터
- 정렬 옵션 (인기순, 가격순, 신상품순, 리뷰순)

### 2.3 장바구니 및 주문 관리 (Cart & Order Management)

#### 2.3.1 장바구니
- 상품 추가/삭제
- 수량 변경
- 찜하기 기능
- 장바구니 저장 (비회원도 가능)
- 대량 주문 계산

#### 2.3.2 주문 프로세스
- 주문서 작성
- 배송지 선택/입력
- 배송 방법 선택 (일반배송, 현장배송, 픽업)
- 결제 방법 선택
- 주문 요약 확인
- 주문 완료

#### 2.3.3 주문 관리
- 주문 내역 조회
- 주문 상태 추적
  - 주문 대기
  - 결제 완료
  - 주문 확인
  - 상품 준비중
  - 배송중
  - 배송 완료
  - 취소/환불
- 주문 취소/수정 (결제 전)
- 주문 상세 내역

#### 2.3.4 배송 관리
- 배송지 정보 관리
- 배송비 계산 (상품 무게/크기, 거리 기반)
- 배송 추적
- 현장 배송 일정 관리
- 배송 완료 확인

### 2.4 결제 시스템 (Payment System)

#### 2.4.1 결제 수단
- 신용카드 결제
- 계좌이체
- 가상계좌 입금
- 기업 신용 구매 (월말 정산)
- 현금 결제 (픽업 시)

#### 2.4.2 결제 프로세스
- 결제 정보 입력
- 결제 승인
- 결제 실패 처리
- 결제 취소/환불
- 결제 내역 관리

#### 2.4.3 결제 보안
- PCI-DSS 준수
- 결제 정보 암호화
- SSL/TLS 통신
- 결제 로그 관리

### 2.5 리뷰 및 평점 (Reviews & Ratings)

#### 2.5.1 상품 리뷰
- 구매 후 리뷰 작성
- 별점 평가
- 리뷰 이미지 첨부
- 리뷰 수정/삭제
- 리뷰 신고 기능

#### 2.5.2 리뷰 관리
- 리뷰 목록 조회
- 리뷰 정렬 (최신순, 평점순, 도움순)
- 리뷰 추천 기능
- 리뷰 검증 (구매 확인)

### 2.6 고객 지원 (Customer Support)

#### 2.6.1 문의 및 고객지원
- 1:1 문의하기
- FAQ 게시판
- 공지사항
- 자주 묻는 질문
- 상품 문의

#### 2.6.2 공지사항
- 시스템 공지
- 이벤트 공지
- 배송 안내
- 공지사항 카테고리별 분류

### 2.7 관리자 기능 (Admin Features)

#### 2.7.1 상품 관리
- 상품 등록/수정/삭제
- 상품 일괄 등록 (엑셀 업로드)
- 상품 이미지 관리
- 재고 관리
- 가격 관리
- 카테고리 관리

#### 2.7.2 주문 관리
- 주문 내역 조회
- 주문 상태 변경
- 주문 취소/환불 처리
- 배송 관리
- 주문 통계

#### 2.7.3 사용자 관리
- 회원 목록 조회
- 회원 정보 관리
- 기업 사용자 인증
- 권한 관리
- 사용자 통계

#### 2.7.4 통계 및 리포트
- 매출 통계
- 상품별 판매 통계
- 사용자 활동 통계
- 재고 통계
- 대시보드

#### 2.7.5 시스템 관리
- 사이트 설정
- 배송비 설정
- 할인 정책 설정
- 이벤트 관리
- 공지사항 관리

## 3. 비기능적 요구사항

### 3.1 성능 요구사항
- 페이지 로딩 시간: 3초 이내
- 동시 접속자: 최소 1,000명 지원
- 데이터베이스 쿼리 최적화
- 이미지 최적화 (CDN 사용)
- 캐싱 전략 적용

### 3.2 보안 요구사항
- HTTPS 적용
- SQL Injection 방지
- XSS 방지
- CSRF 방지
- 비밀번호 암호화 (bcrypt)
- 개인정보 보호 (GDPR 준수)
- 결제 정보 보안

### 3.3 확장성 요구사항
- 모듈화된 아키텍처
- 마이크로서비스 구조 고려 (선택)
- 수평 확장 가능
- 데이터베이스 분리 고려

### 3.4 사용성 요구사항
- 반응형 웹 디자인 (모바일, 태블릿, 데스크톱)
- 직관적인 UI/UX
- 접근성 고려 (WCAG 2.1 AA 수준)
- 다국어 지원 (선택)

### 3.5 호환성 요구사항
- 주요 브라우저 지원 (Chrome, Firefox, Safari, Edge)
- 모바일 브라우저 지원
- 최신 브라우저 버전 2개 이상 지원

### 3.6 데이터 관리 요구사항
- 정기적인 데이터 백업
- 데이터 복구 계획
- 로그 관리
- 데이터 보관 정책

## 4. 기술 스택 제안

### 4.1 백엔드
- **프레임워크**: FastAPI (Python)
- **데이터베이스**: mariadb 최신 버전 (주 데이터베이스), Redis (캐시)
- **인증**: JWT 기반 인증
- **파일 저장**: 내부 로컬 폴더, 추후 확장시 aws s3

### 4.2 프론트엔드
- **상태 관리**: Redux 또는 Zustand
- **UI 라이브러리**: Material-UI 또는 Ant Design
- **프론트 앤드 언어**: TypeScript (버전 5.9.2)
- **프론트 앤드 프레임워크**: Next.js (버전 13.5.6)
- **프론트 앤드 라이브러리**: React (버전 18.3.1)
### 4.2.2- 주요 UI 라이브러리
-  **Radix UI** (핵심 컴포넌트 라이브러리)
사용 중인 컴포넌트:
Dialog, Dropdown Menu, Select, Tabs, Toast, Popover, Checkbox, Switch 등
- **Tailwind CSS** (스타일링), tailwindcss (v3.4.17), tailwind-merge (스타일 병합)
- **shadcn/ui 스타일** (컴포넌트 구조)
- **아이콘 라이브러리** : @heroicons/react (v2.1.5), lucide-react (v0.460.0)
- **기타 UI 관련 라이브러리** @headlessui/react (v2.2.0), framer-motion (애니메이션), recharts (차트), @tanstack/react-table (테이블)

### 4.3 인프라
- **서버**: naver ncloud
- **CI/CD**: GitHub Actions
- **모니터링**: Prometheus, Grafana 추후 개발
- **로깅**: ELK Stack 또는 CloudWatch 추후 적용

### 4.4 외부 연동
- **결제 시스템**: PG사 연동 (이니시스)
- **배송 추적**: 택배사 API 연동
- **SMS/이메일**: 알림톡, 이메일 서비스

## 5. 데이터베이스 설계 주요 엔티티

### 5.1 핵심 엔티티
- **Users**: 사용자 정보
- **Products**: 상품 정보
- **Categories**: 카테고리
- **Orders**: 주문 정보
- **OrderItems**: 주문 상품
- **Cart**: 장바구니
- **Payments**: 결제 정보
- **Shipping**: 배송 정보
- **Reviews**: 리뷰
- **Inventory**: 재고 정보

### 5.2 관계 설정
- 사용자와 주문: 1:N
- 상품과 주문: N:M (OrderItems를 통한 중간 테이블)
- 상품과 카테고리: N:M
- 사용자와 리뷰: 1:N
- 상품과 리뷰: 1:N

### 5.3 빌드 환경
- docker compose 를 이용하여 컨테이너로 관리
